# Reflection — Weather API Dashboard

## a) The concepts I applied

Here in this lab , i learned the use of REST API where this is my first time using this so everything was new new to me.

The API is the waiters in a hotel where the app is able to have a conversation with other servers on the net. In this practical the app will send a request to the online server "OpenWeatherMap" to check on the weather updates of cities where their server will send thier response and its showed in the page

I used 4 API requests which are:

-GET — for gathering weather infromation data when we search for a city

-POST — for saving a location

-PUT — for editing the saved locations

-DELETE — for removing the existing location

I also had to use HTML for creating the pages , CSS for desiging the styles and JavaScript for makeing it interactive or dynamic. The JavaScript's main job is to fetch APIs where it helps send the requests to the other servers.
